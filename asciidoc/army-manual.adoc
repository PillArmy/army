[[chapter-what-is-army]]
== 什么是 Army

[[intro]]
=== Army 是什么

Army 是一个 Java SQL 框架，提供三种核心能力：

. *更好的 SQL 编写方式* —— 忠于 SQL 语义而非模仿其他框架
. *更好的阻塞式 ORM* —— 不做 JPA 式的脏检查、一级缓存、自动 merge/persist
. *方言感知的数据库驱动* —— 精确到数据库版本号

设计哲学：

[quote]
____
Don't create a new world, just map the real world. +
We need standard, we need dialect — it's the real world.
____

[[pipeline]]
=== 核心工作流

[source]
----
Domain 类（@Table 注解）
  → 注解处理器在编译期生成静态元模型类（Stock_）
  → Criteria API 构建 SQL 语句（类型安全，步骤约束）
  → Dialect Parser 根据目标方言生成原生 SQL
  → Session API 执行语句并返回结果
----

[[convention]]
=== 约定

. 以 `_` 开头的接口/类是框架私有 API，代码中不应直接使用
. Domain 类字段名默认映射为数据库列名（驼峰转下划线），可通过 `@Column(name="...")` 覆盖
. 预留字段 `id`、`createTime`、`updateTime`、`version`、`visible` 拥有自动行为（见 <<chapter-advanced>>）

[[chapter-quick-start]]
== 快速开始

[[maven-deps]]
=== 添加依赖

[source,xml]
----
<dependencies>
    <dependency>
        <groupId>io.qinarmy</groupId>
        <artifactId>army-jdbc</artifactId>
        <version>0.6.8-SNAPSHOT</version>
    </dependency>
    <!-- 根据数据库选择方言模块 -->
    <dependency>
        <groupId>io.qinarmy</groupId>
        <artifactId>army-postgre</artifactId> <!-- 或 army-mysql / army-oracle / army-sqlite -->
        <version>0.6.8-SNAPSHOT</version>
    </dependency>
</dependencies>
----

[[annotation-processor]]
=== 配置注解处理器

在包含 Domain 类的 Maven module 中：

[source,xml]
----
<build>
    <plugins>
        <plugin>
            <groupId>org.apache.maven.plugins</groupId>
            <artifactId>maven-compiler-plugin</artifactId>
            <configuration>
                <annotationProcessors>
                    io.army.modelgen.ArmyMetaModelDomainProcessor
                </annotationProcessors>
            </configuration>
        </plugin>
    </plugins>
</build>
----

编译后会在 `target/generated-sources/annotations/` 下生成元模型类。

[[first-domain-class]]
=== 第一个 Domain 类

[source,java]
----
@Table(name = "stock",
       indexes = @Index(name = "uni_stock_exchange_code",
                        fieldList = {"exchange", "code"},
                        unique = true),
       comment = "stock info table")
public class Stock {

    @Generator("ID_GEN")
    @Column(name = "id", comment = "primary key")
    public Long id;

    @Column(name = "create_time", comment = "create time")
    public LocalDateTime createTime;

    @Column(name = "update_time", comment = "update time")
    public LocalDateTime updateTime;

    @Column(name = "version", comment = "optimistic lock version")
    public Integer version;

    @Column(name = "exchange", comment = "stock exchange")
    public String exchange;

    @Column(name = "code", comment = "stock code")
    public String code;

    @Column(name = "name", comment = "stock name")
    public String name;

    @Column(name = "status", comment = "stock status")
    public StockStatus status;       // 枚举会自动映射

    @Column(name = "offer_price", comment = "offer price")
    public BigDecimal offerPrice;

    @Column(name = "listing_date", comment = "listing date")
    public LocalDate listingDate;

    // getter / setter 省略
}

public enum StockStatus {
    OPEN, CLOSE, DELISTING
}
----

[[first-metamodel]]
=== 编译生成的元模型类

`ArmyMetaModelDomainProcessor` 自动生成：

[source,java]
----
public abstract class Stock_ {

    public static final String ID = "id";
    public static final String NAME = "name";
    // ... 其他字段名常量

    // 表引用
    public static final SimpleTableMeta<Stock> T = ...;

    // 字段引用 —— 带完整泛型信息
    public static final PrimaryFieldMeta<Stock, Long> id = ...;
    public static final FieldMeta<Stock, String> code = ...;
    public static final FieldMeta<Stock, String> name = ...;
    public static final FieldMeta<Stock, StockStatus> status = ...;
    public static final FieldMeta<Stock, BigDecimal> offerPrice = ...;
    // ... 其他字段

    static {
        if (fieldSize != 10) {
            throw new RuntimeException("Field count mismatch...");
        }
    }
}
----

[[first-session-factory]]
=== 创建 SessionFactory

[source,java]
----
SessionFactory factory = SyncFactoryBuilder.builder()
    .name("myApp")                          // 必须
    .environment(ArmyEnvironment.getInstance())  // 必须
    .datasource(dataSource)                 // 必须
    .packagesToScan("com.example.domain")   // 必须
    .build();

SyncLocalSession session = ((SyncSessionFactory) factory).localSession();
----

FactoryBuilder 完整方法：

|===
| 方法 | 必须? | 说明
| `name(String)`         | 是 | 应用名称标识
| `environment(ArmyEnvironment)` | 是 | Army 环境实例
| `datasource(DataSource)`       | 是 | JDBC 数据源
| `packagesToScan(String...)`    | 是 | Domain 类所在包路径，逗号分隔
| `classLoader(ClassLoader)`     | 否 | 自定义类加载器
| `loadStaticModel(boolean)`     | 否 | 是否加载静态模型类，默认 true
| `validateOnStartup(boolean)`   | 否 | 启动时校验表结构，默认 false
| `jsonCodec(JsonCodec)`         | 否 | JSON 编解码器
| `xmlCodec(XmlCodec)`           | 否 | XML 编解码器
| `fieldGeneratorFactory(...)`   | 否 | 字段生成器工厂
| `build()`                      | - | 构建 SessionFactory（阻塞模式）
|===

ArmyKey 环境变量：

|===
| Key | 必须? | 说明
| `ArmyKey.DATABASE`     | 是 | 数据库名称（用于匹配方言模块）
| `ArmyKey.DIALECT`      | 否 | 方言枚举，不指定则自动检测
| `ArmyKey.ZONE_OFFSET`  | 否 | 时区偏移量
| `ArmyKey.DDL_MODE`     | 否 | DDL 生成模式：`CREATE` / `NONE` / `DEFAULT`
| `ArmyKey.VISIBLE_MODE` | 否 | 可见性模式：`ONLY_VISIBLE` / `ONLY_INVISIBLE` / `ALL`
| `ArmyKey.SQL_LOG_MODE` | 否 | SQL 日志模式：`ON` / `OFF`
|===

[[first-insert]]
=== 第一次插入

[source,java]
----
Stock stock = new Stock();
stock.setExchange("NASDAQ");
stock.setCode("TSLA");
stock.setName("Tesla Inc.");
stock.setStatus(StockStatus.OPEN);
stock.setOfferPrice(new BigDecimal("250.50"));

Insert stmt = SQLs.singleInsert()
    .insertInto(Stock_.T)
    .values(List.of(stock))
    .asInsert();

long rows = session.update(stmt);
----

生成 SQL（PostgreSQL）：

[source,sql]
----
INSERT INTO stock (id, create_time, update_time, version,
                   exchange, code, name, status, offer_price)
VALUES (?, ?, ?, ?, ?, ?, ?, ?, ?)
----

[[first-query]]
=== 第一次查询

[source,java]
----
Query stmt = SQLs.query()
    .select(Stock_.id, Stock_.code, Stock_.name, Stock_.offerPrice)
    .from(Stock_.T, AS, "s")
    .where(Stock_.exchange.equal("NASDAQ"))
    .orderBy(Stock_.offerPrice, Order.DESC)
    .limit(10)
    .asQuery();

Stock row = session.queryOne(stmt, Stock.class);
List<Stock> list = session.queryList(stmt, Stock.class);
----

生成 SQL：

[source,sql]
----
SELECT s.id, s.code, s.name, s.offer_price
FROM stock AS s
WHERE s.exchange = ?
ORDER BY s.offer_price DESC
LIMIT 10
----

[[chapter-domain-model]]
== Domain 模型定义

[[annotation-overview]]
=== 注解一览

Army 提供以下注解用于 Domain 类定义：

|===
| 注解 | 作用 | 作用域
| `@Table`              | 将类映射为数据库表 | 类
| `@Column`             | 将字段映射为数据库列 | 字段
| `@Generator`          | 指定字段的主键生成策略 | 字段
| `@GeneratorType`      | 注册自定义生成器类 | 类型
| `@Param`              | 生成器参数（运行时常量） | 字段/类型
| `@FieldParam`         | 生成器参数（另一字段引用） | 字段/类型
| `@OverrideParams`     | 覆盖父类生成器参数 | 字段
| `@Mapping`            | 显式指定 MappingType | 字段
| `@Index`              | 表索引定义 | 类/字段
| `@IndexField`         | 索引字段详细配置 | 字段
| `@SortOrder`          | 索引字段排序 | 字段
| `@NullsOrder`         | 索引字段 NULL 排序 | 字段
| `@Inheritance`        | 单表继承映射 | 类
| `@DiscriminatorValue` | 继承判别值 | 类
| `@MappedSuperclass`   | 映射超类（不映射为独立表） | 类
| `@DdlMode`            | DDL 生成行为控制 | 类/字段
| `@Codec`              | 标记性注解（预留） | 类
|===

[[table-annotation]]
=== @Table

将 Java 类映射为数据库表。

属性：

|===
| 属性 | 类型 | 必须? | 说明
| `name`         | String             | 是 | 表名
| `comment`      | String             | 是 | 表注释
| `catalog`      | String             | 否 | 数据库 catalog
| `schema`       | String             | 否 | 数据库 schema
| `indexes`      | @Index[]           | 否 | 索引定义
| `immutable`    | boolean            | 否 | 是否为不可变表（如日志表）
| `allColumnNotNull` | boolean        | 否 | 所有列是否为 NOT NULL
| `ddlMode`      | DdlMode            | 否 | DDL 生成模式
| `tableOptions` | String             | 否 | 表选项（ENGINE 等），方言相关
| `partitionOptions` | String         | 否 | 分区选项
|===

示例：

[source,java]
----
@Table(name = "china_region",
       comment = "china region table",
       indexes = {
           @Index(name = "uni_region_parent_name", fieldList = {"parentCode", "name"}, unique = true),
           @Index(name = "idx_region_name", fieldList = "name")
       })
public class ChinaRegion {
    // ...
}

// 不可变表（插入后不修改）
@Table(name = "stock_quotes",
       immutable = true,
       comment = "stock quotes table")
public class StockQuotes { /* ... */ }
----

[[column-annotation]]
=== @Column

将 Java 字段映射为数据库列。

内置约定（零配置）—— 以下 Java 类型字段无需 `@Column(name = "...")`：

|===
| 类型 | 默认列名 | 默认 JDBC 类型 | 说明
| `long` / `Long`        | 字段名（驼峰转下划线） | BIGINT          | 整数
| `int` / `Integer`      | 同上                   | INTEGER         | 整数
| `String`               | 同上                   | VARCHAR(255)    | 变长字符串
| `BigDecimal`           | 同上                   | NUMERIC(19,4)   | 定点数
| `boolean` / `Boolean`  | 同上                   | BOOLEAN         | 布尔
| `LocalDateTime`        | 同上                   | TIMESTAMP       | 日期时间
| `LocalDate`            | 同上                   | DATE            | 日期
| `LocalTime`            | 同上                   | TIME            | 时间
| `byte[]`               | 同上                   | VARBINARY(255)  | 字节数组
| `BigInteger`           | 同上                   | NUMERIC(38)     | 大整数
| `UUID`                 | 同上                   | UUID            | UUID
|===

属性：

|===
| 属性 | 类型 | 说明
| `name`         | String  | 列名（不指定则自动生成）
| `comment`      | String  | 列注释（不指定则自动生成）
| `notNull`      | boolean | 是否 NOT NULL
| `insertable`   | boolean | 是否参与 INSERT，默认 true
| `updatable`    | boolean | 是否参与 UPDATE，默认 true
| `precision`    | int     | 数字精度
| `scale`        | int     | 小数位数
| `defaultValue` | String  | 默认值（支持占位符）
| `collation`    | String  | 字符排序规则
|===

[[reserved-fields]]
==== 预留字段（自动行为）

以下字段名拥有框架自动行为：

|===
| 字段名 | Java 类型 | 自动行为
| `id`          | Long/long     | 主键，必须存在
| `createTime`  | LocalDateTime | INSERT 时自动注入当前时间
| `updateTime`  | LocalDateTime | UPDATE 时自动注入当前时间
| `version`     | Integer/int   | 乐观锁字段，UPDATE 时自动 version+1 且加 WHERE version=?
| `visible`     | Boolean/boolean | 软删除，SELECT 时自动注入 WHERE visible=TRUE
|===

[[generator-annotation]]
=== @Generator

指定字段的主键生成策略。Army 不支持数据库自增键，所有 ID 由应用生成。

生成器类型：

|===
| 类型 | 说明
| `PRECEDE`  | 前置生成：DAO 层返回 ID（如 Snowflake），INSERT 前确定
| `POST`     | 后置生成：INSERT 后从 ResultSet.getGeneratedKeys() 获取
| `RUNTIME`  | 运行时生成：如数据库 SEQUENCE / AUTO_INCREMENT
| `DEFAULT`  | 框架默认生成器
|===

内置生成器：

|===
| 生成器 | 说明
| `Snowflake8Generator`  | 8 字节 Snowflake（42 位时间戳 + 5 位工作机 + 5 位数据中心 + 12 位序列）
| `Snowflake12Generator` | 12 字节 Snowflake
| `UuidGenerator`        | UUID 生成器
|===

Snowflake 需要配置 `startTime` 参数：

[source,java]
----
@Generator(value = "ID_GEN", startTime = "2020-01-01 00:00:00")
@Column(name = "id", comment = "primary key")
public Long id;
----

自定义生成器通过 `@GeneratorType` 注册：

[source,java]
----
@GeneratorType(MyCustomGenerator.class)
public @interface MyGenerator {
    String value() default "";
}
----

[[mapping-annotation]]
=== @Mapping

显式指定字段的 MappingType，覆盖默认的类型推断：

[source,java]
----
// 指定使用固定长度字符类型
@Mapping("io.army.mapping.SqlCharType")
@Column(name = "file_hash")
public String fileHash;  // CHAR(32) 而非 VARCHAR(255)

// 指定使用 JSONB 类型
@Mapping("io.army.mapping.postgre.PreferredJsonbType")
@Column(name = "tags")
public List<String> tags;
----

解析优先级：`@Mapping` > 默认 Java 类型推断 > 全局 TypeMappingHandler。

[[index-annotation]]
=== @Index

定义表索引。

[source,java]
----
// 类级别索引
@Table(name = "stock",
       indexes = {
           @Index(name = "uni_stock_code", fieldList = {"exchange", "code"},
                  unique = true),
           @Index(name = "idx_stock_name", fieldList = "name",
                  type = IndexType.BTREE)
       })

// 字段级别索引
@Index(name = "idx_stock_status")
@Column(name = "status")
public StockStatus status;
----

索引名约定（自动生成）：

|===
| 类型            | 格式        | 示例
| 唯一索引       | `uni_{table}_{fields}` | `uni_stock_exchange_code`
| 非唯一索引     | `idx_{table}_{fields}` | `idx_stock_name`
|===

`@IndexField` 对每个索引字段进行详细配置：

|===
| 属性 | 说明
| `collation`      | 排序规则
| `operatorClass`  | 操作符类（PostgreSQL）
| `sortOrder`      | 排序方向：ASC / DESC / UNSET
| `nullsOrder`     | NULL 排序：FIRST / LAST / UNSET / LARGE / SMALL
|===

[[ddl-mode]]
=== @DdlMode — DDL 生成控制

控制 DDL 生成行为，用于混合管理（部分表由框架管理，部分外部管理）：

|===
| 模式 | 说明
| `CREATE`   | 框架负责 CREATE TABLE
| `NONE`     | 框架不生成 DDL（表由外部管理，如 Flyway/Liquibase）
| `DEFAULT`  | 跟随全局 `ArmyKey.DDL_MODE` 配置
|===

[source,java]
----
// 表级别：该表不生成 DDL
@Table(name = "external_log",
       ddlMode = DdlMode.NONE,
       comment = "externally managed log table")

// 字段级别：该列不生成 DDL
@Column(name = "legacy_field", ddlMode = DdlMode.NONE)
public String legacyField;

// 全局配置
ArmyEnvironment.getInstance()
    .set(ArmyKey.DDL_MODE, DdlMode.CREATE);
----

[[mapped-superclass]]
=== @MappedSuperclass

映射超类：超类的字段被子实体继承，但超类自身不映射为独立表。

[source,java]
----
@MappedSuperclass
public class BaseDomain {
    @Column(name = "id")
    public Long id;
    @Column(name = "create_time")
    public LocalDateTime createTime;
    @Column(name = "update_time")
    public LocalDateTime updateTime;
}

@Table(name = "stock", comment = "stock table")
public class Stock extends BaseDomain {
    @Column(name = "name")
    public String name;
    // id / createTime / updateTime 自动继承
}
----

[[inheritance-annotation]]
=== @Inheritance

单表继承：父类和所有子类共用一个表，通过 discriminator 列区分类型。

[source,java]
----
@Table(name = "user", comment = "user table")
@Inheritance(discriminator = @Column(name = "user_type"))
public class User {
    @Column(name = "id")
    public Long id;
    @Column(name = "name")
    public String name;
}

@DiscriminatorValue("ADMIN")
public class Admin extends User {
    @Column(name = "admin_level", comment = "admin level")
    public Integer adminLevel;
}

@DiscriminatorValue("MEMBER")
public class Member extends User {
    @Column(name = "member_level", comment = "member level")
    public Integer memberLevel;
}
----

查询、更新、删除子实体时，Army 自动注入 discriminator 条件。

[[properties-placeholder]]
=== TableMeta.properties 占位符

在 `META-INF/army/TableMeta.properties` 中覆盖默认的注解值：

占位符列表：

|===
| 占位符 | 解析时机 | 用途
| `${DEFAULT}`        | 编译时解析 | 固定不变的值（如表名、列名）
| `${RUNTIME}`        | 编译时读取默认，运行时重新读取 | 部署期可覆盖的值
| `${OPTIONAL}`       | 可选值 | 省略时使用注解原始值
| `${DEFAULT_VALUE}`  | 默认值占位 | 列默认值表达式
|===

可覆盖的属性：

|===
| 属性 | 说明
| `_name`         | 表名
| `_comment`      | 表注释
| `_catalog`      | Catalog
| `_schema`       | Schema
| `_type`         | 泛化类型（如 `parent_user` 作为父表名）
| `{field}._name`        | 字段列名
| `{field}._comment`     | 字段注释
| `{field}._precision`   | 字段精度
| `{field}._scale`       | 字段小数位
| `{field}._default`     | 字段默认值
| `{field}.GeneratorType`| 生成器类型
|===

完整配置示例：

[source,properties]
----
# META-INF/army/TableMeta.properties

# 表级配置
stock._name=${DEFAULT}army_stock
stock._comment=${DEFAULT}stock information table v2
stock._schema=${OPTIONAL}public

# 字段级配置
stock.exchange._comment=${DEFAULT}exchange market name
stock.offer_price._precision=${DEFAULT}19
stock.offer_price._scale=${DEFAULT}4

# 生成器覆盖
stock.id.GeneratorType=${DEFAULT}Snowflake8Generator
----

[[chapter-session]]
== Session 与会话管理

[[session-api-overview]]
=== Session API 概览

Army 使用 *两步模式*：先构建 Statement，再通过 Session 执行。

Session 接口的核心方法：

|===
| 方法 | 返回类型 | 说明
| `queryOne(Query, Class<T>)`     | T              | 查询单条记录
| `queryList(Query, Class<T>)`    | List\<T\>      | 查询多条记录
| `queryCount(Query)`             | long           | 查询计数
| `queryExists(Query)`            | boolean        | 查询是否包含数据
| `update(DmlStatement)`          | long           | 执行 INSERT / UPDATE / DELETE，返回影响行数
| `update(BatchUpdate)`           | long           | 批量更新
| `save(List<T>, SaveMode)`       | List\<T\>      | 批量插入 Domain 对象
|===

两步模式的示例：

[source,java]
----
// 步骤 1：构建 Statement（不执行 SQL）
Query stmt = SQLs.query()
    .select(Stock_.code, Stock_.name)
    .from(Stock_.T, AS, "s")
    .where(Stock_.exchange.equal("NASDAQ"))
    .asQuery();

// 步骤 2：通过 Session 执行
List<Stock> results = session.queryList(stmt, Stock.class);
----

[[army-environment]]
=== ArmyEnvironment

`ArmyEnvironment` 是全局环境配置对象，持有所有 `ArmyKey` 到值的映射。

[source,java]
----
ArmyEnvironment env = ArmyEnvironment.getInstance()
    .set(ArmyKey.DATABASE, "postgresql")
    .set(ArmyKey.DIALECT, Dialect.POSTGRE17)
    .set(ArmyKey.ZONE_OFFSET, ZoneOffset.of("+8"))
    .set(ArmyKey.DDL_MODE, DdlMode.NONE)
    .set(ArmyKey.VISIBLE_MODE, Visible.ONLY_VISIBLE)
    .set(ArmyKey.SQL_LOG_MODE, SqlLogMode.ON);
----

[[transaction]]
=== 事务管理

[source,java]
----
SyncLocalSession session = ((SyncSessionFactory) factory).localSession();

// 手动事务
session.startTransaction();
try {
    session.update(insertStmt);
    session.update(updateStmt);
    session.commit();
} catch (Exception e) {
    session.rollback();
    throw e;
}

// 函数式事务
session.transactional(tx -> {
    tx.update(insertStmt);
    tx.update(updateStmt);
    return null;
});
----

Transaction 接口的方法：

|===
| 方法 | 说明
| `commit()`             | 提交事务
| `rollback()`           | 回滚事务
| `setRollbackOnly()`    | 标记为只回滚
| `commitOrRollback()`   | 根据状态提交或回滚
|===

[[basic-select]]
==== 基础查询

[source,java]
----
Query stmt = SQLs.query()

[source]
----

→ orderBy() → limit() → offset() → forUpdate() → asQuery()

[[where-clause]]
==== WHERE 条件

比较运算符：

[source,java]
----
// 等于
Stock_.code.equal("TSLA")
Stock_.code.spaceEqual("TSLA")    // space() 版本

// 不等于
Stock_.status.notEqual(StockStatus.DELISTING)

// 大于 / 小于
Stock_.offerPrice.greaterThan(new BigDecimal("100"))

// BETWEEN
Stock_.offerPrice.between(new BigDecimal("100"), new BigDecimal("500"))

// LIKE
Stock_.name.like("Tesla%")

// IN
Stock_.code.in_(List.of("TSLA", "AAPL", "GOOGL"))

// IS NULL / IS NOT NULL

Stock_.listingDate.isNotNull()

AND / OR 组合：

[source,java]
----
SQLs.query()
    .select(Stock_.code, Stock_.name)
    .from(Stock_.T, AS, "s")
    .where(w -> w
        .and(Stock_.exchange.equal("NASDAQ"))
        .or(Stock_.offerPrice.greaterThan(100), Stock_.code.in_(hotCodes))
    )
    .asQuery();

子查询：

[source,java]

        SQLs.query()
            .select(Order_.userId)

[[expression-api]]
==== Expression API

Army 中，所有 SQL 表达式都实现 `Expression` 或 `TypedExpression` 接口：

[source]
----
Expression → 无类型表达式（如 1*x，不同数据库结果类型不同）
TypedExpression → 带类型信息的表达式（绑定 MappingType）
    ├── IPredicate       → BOOLEAN 类型（WHERE 条件）
    ├── FieldMeta        → 列引用（字段元数据）
    ├── SubQuery         → 子查询表达式
    └── FuncExpression   → 函数表达式
----

关键区别：`Expression` 用于 `SET` 赋值、`SELECT` 列、`ORDER BY` 等场景；
`TypedExpression` 可用于需要推断 SQL 类型的场合（如 CASE WHEN 的 `end()` 方法需要类型）。

[source,java]
----
// Expression: 无类型，可以出现在 SET / SELECT / ORDER BY 中
Expression<?> expr = SQLs.add(Stock_.price, 10);

// TypedExpression: 带类型，可用于 CASE 等需要类型推断的场景
TypedExpression<BigDecimal> typed = Stock_.offerPrice;  // FieldMeta 继承 TypedExpression

// IPredicate 是 TypedExpression<Boolean> 的子类型
IPredicate condition = Stock_.offerPrice.greaterThan(100);
// 可以出现在 CASE WHEN 中
SQLs.cases().when(condition, SQLs.literal("HIGH"));
----

[[on-conflict]]
==== ON CONFLICT（PostgreSQL Upsert）

`army-postgre` 方言支持 `ON CONFLICT`：

[source,java]
----
// DO NOTHING
Insert stmt = Postgres.singleInsert()
    .insertInto(Stock_.T)
    .values(List.of(stock))
    .onConflict(Stock_.code)         // 冲突列
    .doNothing()
    .asInsert();

// DO UPDATE
Insert stmt = Postgres.singleInsert()
    .insertInto(Stock_.T)
    .values(List.of(stock))
    .onConflict(Stock_.code)
    .doUpdate()                      // ON CONFLICT DO UPDATE
    .set(Stock_.name, Postgres.excluded(Stock_.name))
    .set(Stock_.offerPrice, Postgres.excluded(Stock_.offerPrice))
    .where(Stock_.status.notEqual(StockStatus.DELISTING))
    .asInsert();
----

生成 SQL：

[source,sql]
----
INSERT INTO stock (code, name, offer_price, ...)
VALUES (?, ?, ?, ...)
ON CONFLICT (code) DO UPDATE
SET name = EXCLUDED.name,
    offer_price = EXCLUDED.offer_price,
    update_time = NOW()
WHERE stock.status <> ?
----

[[domain-update]]
==== Domain 更新（父子表）

Domain UPDATE 自动处理 `@Inheritance` 父子表：

[source,java]
----
SQLs.domainUpdate()
    .update(Admin_.T, AS, "a")
    .set(Admin_.adminLevel, 9)           // 子表字段
    .set(ParentUser_.name, "Alice")      // 父表字段
    .where(Admin_.id.equal(1L))
    .asUpdate();
----

PostgreSQL 生成 SQL（CTE 双阶段更新）：

[source,sql]
----
WITH a_update_cte AS (
    UPDATE u_admin AS a
    SET admin_level = ?
    FROM u_parent_user AS p_of_a
    WHERE a.id = p_of_a.id
    AND p_of_a.status = 'ADMIN'     -- discriminator 自动注入
    AND p_of_a.visible = true       -- visible 自动过滤
    RETURNING a.id
)
UPDATE u_parent_user AS p_of_a
SET name = ?, update_time = ?, version = p_of_a.version + 1
FROM a_update_cte
WHERE p_of_a.id = a_update_cte.id
----

MySQL 生成 SQL（多表 JOIN）：

[source,sql]
----
UPDATE u_admin AS a
JOIN u_parent_user AS p_of_a ON a.id = p_of_a.id
SET p_of_a.name = ?, a.admin_level = ?,
    p_of_a.update_time = ?, p_of_a.version = p_of_a.version + 1
WHERE p_of_a.id = ?
AND p_of_a.status = 'ADMIN'        -- discriminator 自动注入
AND p_of_a.visible = true          -- visible 自动过滤
----

NOTE: Domain 更新/删除必须在事务中执行，否则抛出 `ChildDmlNoTractionException`。

SET 子句支持运算符：

[source,java]
----
// 除了直接赋值，还支持算术运算
stmt = SQLs.singleUpdate()
    .update(Stock_.T, AS, "s")
    .set(Stock_.offerPrice, SQLs.add(Stock_.offerPrice, new BigDecimal("5")))
    // 生成: SET offer_price = offer_price + ?
    .set(Stock_.version, SQLs.add(Stock_.version, 1))
    // 生成: SET version = version + 1
    .where(Stock_.code.equal("TSLA"))
    .asUpdate();
----

[[jsonb-operators]]
==== PostgreSQL JSONB 运算符

Army 通过 `space()` 方法支持 PostgreSQL 特有的 JSONB 运算符（`?`、`?|`、`?&`、`@>`、`<@` 等），这些运算符无法直接映射为 Java 方法名：

[source,java]
----
// JSONB 包含运算符 @>
SQLs.query()
    .select(User_.ALL)
    .from(User_.T, AS, "u")
    .where(User_.tags.space(PgBiOperator.CONTAINS, SQLs.literal("{\"vip\":true}")))
    .asQuery();

// JSONB 被包含运算符 <@
expression.space(PgBiOperator.CONTAINED_BY, SQLs.literal("[\"admin\"]"))

// JSONB 键存在运算符 ?
expression.space(PgBiOperator.EXISTS, SQLs.literal("key_name"))

// 路径提取运算符 #>
expression.space(PgDualOperator.PATH_GET, SQLs.literal("{nested,key}"))
----

`space()` 方法接受 `DualOperator`（返回 `Expression`）或 `BiOperator`（返回 `IPredicate`），支持链式调用。

[[chapter-advanced]]
== 高级特性

[[visible-soft-delete]]
=== 软删除（Visible）

Army 通过 `visible` 字段实现零注解零配置软删除：

[source,java]
----
public class User {
    @Column(name = "visible", comment = "visible flag")
    public Boolean visible;  // true 表示可见，false 表示已删除
}
----

三种可见性模式：

|===
| 模式 | 效果
| `Visible.ONLY_VISIBLE`    | SELECT 自动注入 `WHERE visible = TRUE`
| `Visible.ONLY_INVISIBLE`  | SELECT 自动注入 `WHERE visible = FALSE`
| `Visible.ALL`             | 不注入任何条件
|===

配置方式：

[source,java]
----
// 全局配置
ArmyEnvironment env = ArmyEnvironment.getInstance()
    .set(ArmyKey.VISIBLE_MODE, Visible.ONLY_VISIBLE);
----

查询时的 `visible` 过滤完全由框架 parser 在 SQL 生成层面注入，不依赖 AOP、拦截器或回调。

[[optimistic-lock]]
=== 乐观锁（Version）

通过字段名 `version` 识别（无需注解）：

[source,java]
----
public class Stock {
    @Column(name = "version")
    public Integer version;  // 框架自动管理
}
----

自动行为：

. UPDATE 时自动将 `version` 加 1：`SET version = version + 1`
. WHERE 子句自动添加：`AND version = ?`
. 执行器检测更新行数为 0 时，抛出 `OptimisticLockException`

[[auto-timestamp]]
=== 自动时间戳

`createTime` 和 `updateTime` 字段名由框架自动管理：

* `createTime`：INSERT 时自动注入 `NOW()`（方言相关）
* `updateTime`：UPDATE 时自动注入 `NOW()`

无需任何注解，只需字段名匹配即可。

[[parent-child]]
=== 父子表（@Inheritance）

当使用 `@Inheritance` 定义单表继承时，UPDATE / DELETE 子实体自动生成正确的 SQL：

. PostgreSQL：使用 CTE（公共表表达式）先更新子表，再更新父表
. MySQL：使用多表 JOIN UPDATE / DELETE 语法

父子表更新在事务中先子后父执行。如果不在事务中，会抛出 `ChildDmlNoTractionException`。

[source,java]
----
// 更新 Admin（User 的子类）时自动处理父子表
SQLs.domainUpdate()
    .update(Admin_.T, AS, "a")
    .set(Admin_.adminLevel, 9)
    .set(ParentUser_.name, "Alice Updated")
    .where(Admin_.id.equal(1L))
    .asUpdate();
----

生成 SQL（PostgreSQL）：

[source,sql]
----
-- CTE 先更新子表
WITH a_update_cte AS (
    UPDATE u_admin AS a
    SET admin_level = ?
    FROM u_parent_user AS p_of_a
    WHERE a.id = p_of_a.id
    AND p_of_a.status = 'ADMIN'    -- discriminator 自动注入
    AND p_of_a.visible = true      -- visible 自动注入
    RETURNING a.id
)
-- 再更新父表
UPDATE u_parent_user AS p_of_a
SET name = ?, update_time = ?, version = p_of_a.version + 1
FROM a_update_cte
WHERE p_of_a.id = a_update_cte.id
----

[[snowflake-generator]]
=== 主键生成器

Army 不支持数据库自增键。所有 ID 由应用端生成。

Snowflake8Generator 配置：

[source,java]
----
@Generator(value = "ID_GEN", startTime = "2020-01-01 00:00:00")
@Column(name = "id", comment = "primary key")

----

8 字节雪花算法结构：42 位时间戳 + 5 位工作机 + 5 位数据中心 + 12 位序列号。

[[chapter-types]]
== 类型系统

[[mappingtype-overview]]
=== MappingType 体系

`MappingType` 是 Army 的类型映射核心 —— 每一个 Java 类型到每一个 SQL 类型都有一个 `MappingType` 实例。

[source,java]
----
public sealed interface MappingType
    permits AbstractMappingType, ArrayMappingType {

    Class<?> javaType();              // Java 类型
    String map(ServerMeta meta);     // 输出 SQL 类型名（如 "VARCHAR(255)"）
    Object beforeBind(...);          // Java → JDBC 转换
    Object afterGet(...);           // JDBC → Java 转换
}
----

体系结构：

[source]
----
MappingType (sealed)
├── _ArmyBuildInType (sealed, package guard)
│   ├── SqlVarcharType
│   ├── SqlIntegerType
│   ├── SqlBigIntType
│   ├── SqlTimestampType
│   └── ... (50+ 内置类型)
├── UserMappingType (non-sealed, 用户扩展点)
└── MappingTypeType (自反元类型)
----

[[default-type-inference]]
=== 默认类型推断

Army 根据 Java 字段类型自动推断 MappingType：

|===
| Java 类型 | MappingType | SQL 类型
| `String`             | `SqlVarcharType(255)`        | VARCHAR(255)
| `long` / `Long`      | `SqlBigIntType`              | BIGINT
| `int` / `Integer`    | `SqlIntegerType`             | INTEGER
| `float` / `Float`    | `SqlRealType`                | REAL
| `double` / `Double`  | `SqlDoubleType`              | DOUBLE
| `short` / `Short`    | `SqlSmallIntType`            | SMALLINT
| `byte` / `Byte`      | `SqlTinyIntType`             | TINYINT
| `boolean` / `Boolean`| `SqlBooleanType`             | BOOLEAN
| `BigDecimal`         | `SqlNumericType(19,4)`       | NUMERIC(19,4)
| `BigInteger`         | `SqlNumericType(38)`         | NUMERIC(38)
| `LocalDateTime`      | `SqlLocalDateTimeType`       | TIMESTAMP
| `LocalDate`          | `SqlLocalDateType`           | DATE
| `LocalTime`          | `SqlLocalTimeType`           | TIME
| `OffsetDateTime`     | `SqlOffsetDateTimeType`      | TIMESTAMP WITH TZ
| `ZonedDateTime`      | `SqlZonedDateTimeType`       | TIMESTAMP WITH TZ
| `Instant`            | `SqlInstantType`             | TIMESTAMP
| `UUID`               | `SqlUUIDType`                | UUID
| `byte[]`             | `SqlVarBinaryType`           | VARBINARY(255)
| `BitSet`             | `SqlBitSetType`              | 方言相关
| `ZoneId`             | `SqlZoneIdType`              | VARCHAR
|===

[[type-mapping-handler]]
=== TypeMappingHandler 类型解析

字段类型到 SQL 类型的解析分三步：

. 检查 `@Mapping` 注解：如果指定了 MappingType 全限定名，直接使用
. 类型推断：根据 Java 字段类型查找默认 MappingType
. `MappingType.map(ServerMeta)`：将 MappingType 转换为 SQL 类型字符串

`TypeMappingHandler` 是一个 sealed 接口，由 `TypeMappingHandlerSupport` 提供默认实现。

[[enum-mapping]]
=== 枚举映射

Army 支持三种枚举映射方式：

[source,java]
----
// 1. 普通枚举 → 映射 name()，DDL 生成 CHECK 约束
public enum StockStatus { OPEN, CLOSE, DELISTING }

// 2. TextEnum → 映射 String label()
public enum OrderStatus implements TextEnum {
    PENDING("pending"), PROCESSING("processing"), DONE("done");
    private final String text;
    @Override public String text() { return text; }
}

// 3. CodeEnum → 映射 int code()
public enum BankAccountType implements CodeEnum {
    SAVINGS(1), CHECKING(2), CREDIT(3);
    private final int code;
    @Override public int code() { return code; }
}
----

数据库端枚举（PostgreSQL Native ENUM）：

[source,java]
----
@DefinedType(name = "stock_status_enum", category = TypeCategory.ENUM)
public enum StockStatusWithPgEnum {
    OPEN, CLOSE, DELISTING
}
----

[[custom-mappingtype]]
=== 自定义 MappingType

. 选择一个基类继承（以下任选其一）：

|===
| 基类 | 适用场景
| `UserMappingType`     | 单值自定义类型
| `UserArrayType`       | 数组自定义类型
| `_ArmyCoreBuildInType`| 需要更底层控制
|===

. 实现合适的 `Sql*` 标记接口，声明该类型在 SQL 中的角色：

|===
| 标记接口 | 说明
| `MappingType.SqlNumericType`  | 数字类型
| `MappingType.SqlCharacterType`| 字符类型
| `MappingType.SqlDateTimeType` | 日期时间类型
| `MappingType.SqlBinaryType`   | 二进制类型
| `MappingType.SqlBooleanType`  | 布尔类型
| `MappingType.SqlUUIDType`     | UUID 类型
| `MappingType.SqlJsonType`     | JSON 类型
| `MappingType.SqlXmlType`      | XML 类型
|===

. 实现核心方法：

[source,java]
----
// 自定义货币类型（完整实现）
public class CurrencyType extends UserMappingType
        implements MappingType.SqlNumericType {

    public static CurrencyType from(Class<?> javaType) {
        return new CurrencyType(javaType);
    }

    public CurrencyType(Class<?> javaType) {
        super(javaType);     // 如 BigDecimal.class
    }

    @Override
    public String map(ServerMeta meta) {
        return "NUMERIC(19,6)";
    }

    @Override
    public Object beforeBind(ServerMeta meta, Object value) {
        return value;
    }

    @Override
    public Object afterGet(ResultSet rs, int index,
                            DataAccess access, Session session,
                            ServerMeta meta) throws SQLException {
        return rs.getBigDecimal(index);
    }

    @Override
    public int hashCode() { return CurrencyType.class.hashCode(); }

    @Override
    public boolean equals(Object obj) { return obj instanceof CurrencyType; }
}
----

. 使用：

[source,java]
----
@Mapping("com.example.CurrencyType")
@Column(name = "total_fee")
public BigDecimal totalFee;
----

[[faq]]
=== A. 常见问题

. *Army 和 MyBatis 有什么区别？* +
Army 是纯 Java API（无 XML/模板），编译期类型安全，阶梯构建器约束 SQL 语法顺序。MyBatis 用 XML/注解写 SQL 字符串。

. *Army 和 JOOQ 有什么区别？* +
两个都是类型安全的 Java SQL 框架。关键区别：Army 不做代码生成（用注解处理器生成静态元模型），类型以 sealed 体系精确分型，space() 做子句切换而非方法爆炸，方言版本精确到版本号。

. *Army 和 JPA/Hibernate 有什么区别？* +
Army 不做 ORM（不做脏检查、一级缓存、自动 merge/persist）。开发者显式控制 INSERT 还是 UPDATE。适合对 SQL 有精确控制需求的场景。

. *支持哪些数据库？* +
PostgreSQL（最完整）、MySQL、Oracle、SQLite。

. *怎么开启 SQL 日志？* +
`ArmyEnvironment.getInstance().set(ArmyKey.SQL_LOG_MODE, SqlLogMode.ON)`

. *字段名必须是驼峰命名吗？* +
不必须。驼峰自动转下划线是默认行为，通过 `@Column(name = "...")` 可以指定任意列名。

. *支持自增主键吗？* +
不支持。Army 设计哲学是应用端生成 ID（Snowflake / UUID），保证分布式一致性。
