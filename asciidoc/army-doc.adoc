== 前言

=== Army 框架简介

Army 是一个更好的 Java SQL 框架，提供以下核心能力：

* 一种更好的在 Java 中编写 SQL 的方式
* 一种更好的阻塞式 ORM 框架
* 一种更好的响应式 ORM 框架
* 一种更高层次的数据库驱动

=== 设计理念

1. **不创造新世界，只映射真实世界** - 尊重数据库的标准和方言差异
2. **我们需要标准，也需要方言** - 这是真实世界的写照
3. **我们需要阻塞方式，也需要响应式方式** - 适应不同的应用场景

=== 项目约定

* 以下划线开头的接口或类是 Army 框架的私有接口或类

== 核心架构

=== 模块划分

Army 框架采用模块化设计，包含以下核心模块：

==== army-core（核心模块）

Army 框架的核心实现，包含：

* Criteria API 标准实现
* SQL 解析和生成
* 元数据管理
* 会话管理
* 事务管理
* 类型映射
* 执行器支持

==== army-annotation（注解模块）

提供领域对象映射注解：

* `@Table` - 表映射
* `@Column` - 字段映射
* `@Inheritance` - 继承策略
* `@MappedSuperclass` - 映射超类
* `@Generator` - 字段生成器
* `@Index` - 索引定义
* `@Mapping` - 类型映射
* `@DiscriminatorValue` - 鉴别器值

==== army-modelgen（代码生成模块）

注解处理器实现，自动生成静态元模型类：

* `ArmyMetaModelDomainProcessor` - 主要处理器
* `AnnotationHandler` - 注解处理逻辑
* `SourceCodeCreator` - 源代码生成器
* `MappingMode` - 映射模式（简单、父表、子表）
* `IndexMode` - 索引模式

==== army-mysql（MySQL 方言模块）

MySQL 数据库特定实现：

* MySQL Criteria API 扩展
* MySQL 方言解析器
* MySQL 特有函数和操作符
* MySQL DDL 解析

==== army-postgre（PostgreSQL 方言模块）

PostgreSQL 数据库特定实现：

* PostgreSQL Criteria API 扩展
* PostgreSQL 方言解析器
* PostgreSQL 特有功能（RETURNING、ON CONFLICT 等）
* PostgreSQL 复合类型支持
* PostgreSQL 数组类型支持

==== army-oracle（Oracle 方言模块）

Oracle 数据库特定实现（待完善）

==== army-sqlite（SQLite 方言模块）

SQLite 数据库特定实现

==== army-jdbc（JDBC 执行器模块）

基于 JDBC 的执行器实现：

* `JdbcExecutor` - JDBC 执行器
* `JdbcExecutorFactory` - 执行器工厂
* `MySQLExecutor` - MySQL 执行器
* `PostgreExecutor` - PostgreSQL 执行器
* `SQLiteExecutor` - SQLite 执行器

==== army-spring（Spring 集成模块）

Spring 框架集成：

* `ArmySessionFactoryBeanSupport` - SessionFactory Bean 支持
* `SpringArmyEnvironment` - Spring 环境配置
* 同步模式集成（sync 包）
* 响应式模式集成（reactive 包）
* Spring 事务管理器集成

==== army-sync（同步模式模块）

阻塞式同步 API 实现

==== army-reactive（响应式模式模块）

响应式 API 实现（基于 Reactor）

==== army-struct（基础结构模块）

基础数据结构和工具：

* 语言支持（lang 包）
* POJO 工具
* 数据结构

==== army-spring-ai-model-chat-memory（Spring AI 聊天记忆模块）

Spring AI 聊天记忆的 Army 实现

==== army-spring-ai-vector-store（Spring AI 向量存储模块）

Spring AI 向量存储的 Army 实现

== 注解系统

=== @Table 注解

表级别映射注解。

==== 属性说明

* `name` - 表名（必填）
* `comment` - 表注释（必填）
* `catalog` - 目录名（可选）
* `schema` - 模式名（可选）
* `indexes` - 索引定义数组

==== 使用示例

[source,java]
----
@Table(name = "user_info", schema = "public", comment = "用户信息表")
public class UserInfo {
    // ...
}
----

=== @Column 注解

字段级别映射注解。

==== 属性说明

* `name` - 列名（可选，默认使用字段名）
* `comment` - 列注释
* `nullable` - 是否可为空
* `precision` - 精度
* `scale` - 小数位数
* `defaultValue` - 默认值
* `updatable` - 是否可更新
* `insertable` - 是否可插入
* `updateMode` - 更新模式
* `length` - 长度

==== 使用示例

[source,java]
----
@Column(name = "user_name", precision = 50, nullable = false, comment = "用户名")
private String userName;
----

=== @Inheritance 注解

继承策略注解，用于表继承映射。

==== 属性说明

* `value` - 鉴别器字段名

==== 使用示例

[source,java]
----
@Inheritance("regionType")
public class ChinaRegion<T extends ChinaRegion<T>> {
    // ...
}
----

=== @Generator 注解

字段值生成器注解。

==== 属性说明

* `type` - 生成器类型（PRECEDE、POST、RUNTIME、DEFAULT）
* `value` - 生成器类名
* `params` - 生成器参数

==== 生成器类型

* `PRECEDE` - 执行前生成
* `POST` - 执行后生成（数据库生成）
* `RUNTIME` - 运行时生成
* `DEFAULT` - 默认生成

==== 使用示例

[source,java]
----
@Generator(type = GeneratorType.PRECEDE, 
           value = "io.army.generator.snowflake.Snowflake8Generator",
           params = {@Param(name = "startTime", value = "1580224449498")})
@Column
private Long id;
----

=== @Index 注解

索引定义注解。

==== 属性说明

* `name` - 索引名
* `fieldList` - 字段列表
* `unique` - 是否唯一索引
* `comment` - 索引注释

==== 使用示例

[source,java]
----
@Index(name = "idx_user_email", fieldList = "email", unique = true)

映射超类注解，用于定义可继承的映射。

鉴别器值注解，用于继承映射。

标准 SQL 语句构建的入口类。

==== 查询语句（SELECT）

[source,java]
----
final Select stmt;
stmt = SQLs.query()
        .select(ChinaRegion_.id, ChinaRegion_.name, ChinaRegion_.regionGdp)
        .from(ChinaRegion_.T, AS, "c")
        .where(ChinaRegion_.name.equal(SQLs::param, "曲境"))
        .and(ChinaRegion_.createTime::less, SQLs::literal, LocalDateTime.now().minusDays(1))
        .limit(SQLs::literal, 1)
        .asQuery();
----

=== CTE（公共表表达式）

[source,java]
----
final Insert stmt;
stmt = Postgres.singleInsert()
        .with("parent").as(s -> s
                .insertInto(ChinaRegion_.T)
                .values(provinceList)
                .returning(ChinaRegion_.id)
                .asReturningInsert()
        )
        .comma("parent_row_id").as(s -> s
                .select(ss -> ss
                        .space(Postgres.rowNumber().over().as("rowNumber"))
                        .comma(SQLs.refField("parent", ChinaRegion_.ID))
                )
                .from("parent")
                .asQuery()
        )
        .insertInto(ChinaProvince_.T)
        .values(provinceList)
        .asInsert();
----

} catch (Exception e) {
    session.rollbackToSavepoint("sp1");
    session.update(stmt3);
    session.commit();
}
----

== 最佳实践

=== 领域类设计

1. 使用泛型支持继承映射
2. 为所有字段添加注释
3. 合理使用保留字段（id、createTime、updateTime、version、visible）
4. 正确配置生成器

=== Criteria API 使用

1. 优先使用标准 Criteria API
2. 仅在需要特定数据库功能时使用方言 API
3. 合理使用参数绑定 vs 字面量绑定
4. 注意 CTE 和子查询的性能影响

=== 事务管理

1. 使用声明式事务（@Transactional）
2. 合理设置事务隔离级别
3. 注意长事务的性能影响
4. 使用保存点进行细粒度控制

=== 性能优化

1. 使用批量操作代替循环单条操作
2. 合理使用索引
3. 避免 N+1 查询问题
4. 使用适当的 fetch 策略

=== 类型映射

1. 优先使用内置类型映射
2. 谨慎使用自定义类型映射
3. 正确配置 JSON/XML 编解码器
4. 注意时区处理

== 常见问题

=== 注解处理器未生成元模型类

检查： 1. 是否正确配置了注解处理器 2. 领域类是否使用了 `@Table` 注解 3. 编译时是否包含 army-annotation 依赖

=== Snowflake 生成器警告

问题：多个领域类使用相同的 startTime 参数

解决方案： 1. 使用独立的 startTime 配置 2. 通过配置关闭警告：`-Aarmy.snowflakeStartTimeWarning=false`

=== 事务异常

问题：`IllegalTransactionStateException`

解决方案： 1. 检查是否正确开启了事务 2. 检查事务传播行为配置 3. 确保在事务上下文中执行数据库操作

=== 类型映射错误

问题：类型转换异常

解决方案： 1. 检查字段类型与数据库类型是否匹配 2. 配置正确的 MappingType 3. 检查 JSON/XML 编解码器配置
